ApogeeMobi Offers SDK

Universal browser-side integration for the ApogeeMobi tracking platform. Works on any website. Two lines of HTML.

v1.0 vanilla JS ~10 KB gzipped no dependencies

Quick start

Add this to your site's <head>:

<script src="https://track.apogeemobi.com/sdk/v1/apogee.js"
        data-api-key="YOUR_API_KEY_HERE"
        async></script>

Then put a placeholder anywhere in your page body where offers should appear:

<div data-apg-placement="top-10-sportsbooks"></div>

That's the entire integration. The SDK auto-detects the div on page load, calls /api/v1/serve, and renders the offers inside.

Get your API key from the platform admin: track.apogeemobi.com → API Access → New API Key under the publisher that owns this site's inventory.

Examples by platform

Static HTML / Hostinger / FTP-only sites

Open your HTML file, paste the script tag in <head>, and the placeholder div wherever you want offers. Save. Done.

<!DOCTYPE html>
<html><head>
  <script src="https://track.apogeemobi.com/sdk/v1/apogee.js"
          data-api-key="apg_xxx" async></script>
</head><body>
  <h1>Best Sportsbooks 2026</h1>
  <div data-apg-placement="hp-top10"></div>
</body></html>

Astro

In src/layouts/Layout.astro:

<head>
  <script
    src="https://track.apogeemobi.com/sdk/v1/apogee.js"
    data-api-key={import.meta.env.PUBLIC_APG_KEY}
    async
  />
</head>

Then in any page: <div data-apg-placement="hp-top10"></div>

Plain PHP (not WordPress)

Add to your common header.php or template:

<script
  src="https://track.apogeemobi.com/sdk/v1/apogee.js"
  data-api-key="<?= htmlspecialchars(APG_KEY) ?>"
  async></script>

Next.js / React

In pages/_document.js or app router:

<Script
  src="https://track.apogeemobi.com/sdk/v1/apogee.js"
  data-api-key={process.env.NEXT_PUBLIC_APG_KEY}
  strategy="afterInteractive"
/>

Shopify / Wix / Squarespace

Use the "custom HTML" or "code injection" feature. Paste the script tag once in site-wide settings.

Hugo / Gatsby / Jekyll

Add to your base layout partial. Same pattern as static HTML.

Configuration options

On the <script> tag

AttributeDefaultDescription
data-api-keyrequiredGenerated in admin → API Access
data-api-basederived from script srcOverride the platform URL
data-countryauto (backend GeoIP)Force a country code
data-cache-ttl300Seconds. localStorage cache lifetime
data-debugoffEnable verbose console logging

On the placement <div>

AttributeDefaultDescription
data-apg-placementrequiredThe placement slug from the platform admin
data-apg-limitplacement defaultMax offers to render
data-apg-show-rank10 to hide rank in tables
data-apg-countryinheritsPer-placement country override
data-apg-labelcreative's CTAOverride CTA label for cta placements

Placement types

  • comparison_table — multi-row table with rank, brand, offer, CTA
  • offer_card — single offer in a card layout
  • cta — just the button
  • banner, interstitial — card layout (specialized rendering coming)

How it works

  1. Visitor loads your page.
  2. SDK downloads (~10 KB gzipped, browser-cached).
  3. SDK auto-detects [data-apg-placement] divs.
  4. For each, calls GET /api/v1/serve?placement_slug=...&country=... with the API key.
  5. Response cached in localStorage for 5 minutes per visitor.
  6. SDK injects the offer HTML into the div.
  7. Visitor clicks an offer → standard <a href> through your tracker → advertiser.

SEO considerations

The SDK renders offers in the visitor's browser. Your article HTML — which is what ranks — is server-rendered as usual by your site.

Google can execute JavaScript, but isn't guaranteed to on every crawl. If the offer brands appearing in your SERP snippet matters, prefer a server-side integration: fetch /api/v1/serve from your site's backend and bake the offers into the HTML before sending it to the visitor.

For most affiliate review sites, the SDK is the right choice — your article content carries the SEO weight; offers are monetization, not ranking content.

Debugging

Add data-debug to the script tag:

<script src=".../apogee.js" data-api-key="..." data-debug async></script>

Open your browser console for SDK initialization, cache hits/misses, and errors.
