🔧 The 5 specific changes

  1. Fix 1 — Drop 4 columns from default view. Vertical, Installs, EPC, Revenue hidden behind a "+ Show more columns" toggle. They're still accessible, just not always taking up screen width. Most of those columns were empty for most rows anyway.
  2. Fix 2 — Sticky right edge for Status + Actions. Even if Show More is on and the table goes wider than the viewport, the Status badge and Edit/Clone/Archive buttons stay pinned to the right. position: sticky; right: 0 with a subtle box-shadow on the left edge so users can see they're pinned.
  3. Fix 3 — Merge Adv. Payout + Pub. Payout into one Payout column. Shown as $0.70 / $1.00 CPA — publisher payout in green (what we pay), advertiser payout in muted color (what we charge). The payout type (CPA/CPI/CPL) becomes a colored badge instead of inline text. Net effect: 2 columns → 1.
  4. Fix 4 — Wider Campaign name column with domain on second line. Currently long names like "ID - xlWin - SOI - USD | DO NOT use these links…" get truncated mid-warning. With more horizontal room AND the domain on a second line, the warning stays readable. table-layout: fixed + flex column width.
  5. Fix 5 — Color-coded row left borders by status. 3px solid left border per row: green = active, amber = paused, gray = archived. Scannable at a glance — you can see which campaigns are running without reading the Status column.

⚙ What doesn't change

  • Backend untouched — no schema change, no query change, no API change. Same campaign rows, same data, just rendered differently.
  • All 14 columns are still available — 4 of them are just behind the "Show more" toggle by default. Click it and they appear inline.
  • Sorting / filtering behavior — unchanged. The existing channel / archived / featured filters at the top of the page work the same.
  • The Pending Access Requests box at the top — unchanged.
  • Numbering — # column still shows sequential 1..N descending (already deployed yesterday).
  • URLs — clicking a campaign name still goes to /campaigns/{id}. Edit goes to /campaigns/{id}/edit. Same as today.
