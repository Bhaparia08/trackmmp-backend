1. Welcome

This guide tells you everything you need to send traffic to Apogeemobi offers and get paid for conversions. Cover-to-cover takes ~20 minutes; the 5-minute setup below is enough to start running.

Apogeemobi is a Mobile Measurement Partner (MMP) and affiliate tracking platform. You'll:

  1. Get a tracking link from your account manager (one per campaign)
  2. Append your own click ID + source identifier to that link
  3. Send traffic to it (it 302-redirects to the advertiser's destination)
  4. Receive postbacks from us back to your URL when conversions happen
In a hurry?

Skip to section 2 — 5-minute setup. The rest of this guide is reference material you can come back to.

2. 5-minute setup

Get your tracking link

Your account manager hands you a URL like:
https://track.apogeemobi.com/track/click/yRiop9cOjpI0?pid=38

The pid=38 is YOUR publisher ID (already filled in). The token yRiop9cOjpI0 identifies the campaign you're promoting.

Append your click ID

For every click you send, add a unique clickid parameter — this is YOUR own identifier so postbacks can be tied to that specific click on your side:

https://track.apogeemobi.com/track/click/yRiop9cOjpI0?pid=38&clickid=abc123xyz

If you have multiple traffic sources, also append sub1=facebook (or whatever names your sources).

Send traffic

When a user opens that URL, they're 302-redirected to the advertiser. Your click is recorded against your pid.

Set your postback URL (one-time)

In your publisher portal → Postback Settings, enter the URL where you want us to notify you of conversions, with macros for the dynamic parts:

https://your-tracker.com/postback?clickid={click_id}&payout={payout}&sub1={sub1}

The {click_id} placeholder gets replaced with the same clickid you sent on the original click — so you can attribute the conversion.

Done

Conversions fire automatically. View earnings in your portal → Reports.

3. Your tracking link

Every tracking link follows this shape:

https://track.apogeemobi.com/track/click/CAMPAIGN_TOKEN?pid=YOUR_PID&clickid=YOUR_CLICK_ID
PartWhat it isWho provides it
CAMPAIGN_TOKEN12-character ID for the specific campaignAccount manager (one per campaign)
YOUR_PIDNumeric publisher IDApogeemobi (shown in your portal, e.g. 38)
YOUR_CLICK_IDYour own unique ID for THIS clickYour traffic source / tracker generates it
Important

The CAMPAIGN_TOKEN stays the same per campaign. The YOUR_CLICK_ID must be UNIQUE per click — otherwise multiple conversions can't be distinguished. Most trackers auto-generate this; if you're rolling your own, use a UUID or timestamp+random.

5. Source breakdown — sub1 through sub10

If you push traffic from multiple sources (Facebook, TikTok, Google Ads, native widgets, etc.) and want your conversion reports broken down by source, use sub1...sub10.

Common convention

SlotTypical useExample value
sub1Traffic source / network namefacebook · tiktok · google · taboola
sub2Campaign / ad set namefall_promo · retargeting_q4
sub3Ad / creative IDvid_15s_blue · banner_300x250
sub4Placement / widgetfeed · stories · widget_xyz
sub5Audience / segmentlookalike_2pct · retarget_30d
sub6 – sub10Free-form — whatever you want—
AppsFlyer publishers

We also accept af_sub1...af_sub5 as aliases (same fields, AppsFlyer convention). Either works. sub6–sub10 have no af_ alias.

Site identifier (separate from sub-params)

If you run offers across multiple of your own sites, also add af_siteid (alias site_id) to identify which site:

?pid=38&clickid=abc123&sub1=facebook&af_siteid=top10betting.us

6. Mobile app vs Web — two different patterns

Web traffic

Browser-based offers

Insurance, finance, BNPL comparisons, e-commerce — clicks resolve to a web destination.

Required params

?pid=38
&clickid={CLICK_ID}
&sub1={SOURCE}

No advertising_id needed — web campaigns skip the GAID/IDFA fraud check.

Mobile traffic

App install offers

App campaigns (CPI) — clicks resolve to App Store / Play Store, conversion = install.

Required params

?pid=38
&clickid={CLICK_ID}
&sub1={SOURCE}
&advertising_id={GAID_OR_IDFA}

Missing advertising_id → click blocked by fraud rule (mobile campaigns only).

8. How postbacks work

A postback is an HTTP request the platform fires to YOUR URL when a conversion happens. It's how you find out which clicks converted.

┌─────────────────────────────────────────────────────────────────────┐
│  1. User clicks your tracking URL with ?clickid=abc123              │
│       ↓                                                              │
│  2. Apogeemobi records click, 302-redirects user to advertiser      │
│       ↓                                                              │
│  3. User converts on advertiser site (install / signup / purchase)  │
│       ↓                                                              │
│  4. Advertiser fires conversion to Apogeemobi (S2S or pixel)        │
│       ↓                                                              │
│  5. Apogeemobi fires YOUR postback URL, substituting macros:        │
│         https://your-tracker.com/cvr?clickid=abc123&payout=2.50      │
│       ↓                                                              │
│  6. Your tracker receives it and credits the conversion to abc123   │
└─────────────────────────────────────────────────────────────────────┘

9. Setting up your postback URL

In your publisher portal → Postback Settings (or via API), set ONE postback URL. It will be fired for every conversion attributed to your pid.

Template format

Use {macro_name} placeholders for the dynamic parts. The platform substitutes real values when firing.

Minimum useful postback

https://your-tracker.com/postback
  ?clickid={click_id}
  &payout={payout}
  &status={status}

Recommended (more attribution detail)

https://your-tracker.com/postback
  ?clickid={click_id}
  &payout={payout}
  &currency={currency}
  &event={event_name}
  &sub1={sub1}
  &sub2={sub2}
  &country={country}
  &platform={platform}

Full power (geo + device + retargeting)

https://your-tracker.com/postback
  ?clickid={click_id}
  &payout={payout}
  &currency={currency}
  &event={event_name}
  &goal_id={goal_id}
  &sub1={sub1}&sub2={sub2}&sub3={sub3}&sub4={sub4}&sub5={sub5}
  &country={country}
  &region={region}
  &city={city}
  &device_type={device_type}
  &os={os}
  &browser={browser}
  &gaid={gaid}
  &is_retargeting={is_retargeting}

11. Postback status codes & retries

StatusWhenWhat happens to your postback?
attributedClick+event matched a live click within lookback windowPostback fires immediately
duplicateSame click+event already attributedPostback fires with {status}=duplicate (your tracker should ignore)
rejectedClick ID unknown, lookback expired, or fraud rule blockedPostback may fire with {status}=rejected depending on your settings

Retry behavior

If your postback endpoint returns non-2xx or times out, we retry with exponential backoff:

AttemptDelay
1immediate
2+30 seconds
3+2 minutes
4+10 minutes
5+1 hour

After 5 failed attempts the postback is marked as failed in your dashboard. To make retries reliable, your endpoint must always respond with 200 OK within 10 seconds.

13. Pixel impression (image tag, no JS)

For impression tracking on pages that can't run JS:

<img
  src="https://track.apogeemobi.com/pixel.gif?campaign_token=CAMPAIGN_TOKEN&pid=YOUR_PID"
  width="1" height="1"
  style="display:none">

Returns a 42-byte transparent gif. Counts as an impression — used for frequency capping and view-through attribution.

15. S2S server-to-server postback

If you (or your network) want to fire conversions to us instead of the advertiser firing — use S2S:

POST https://track.apogeemobi.com/api/s2s/event
Headers:
  authentication: YOUR_S2S_TOKEN
  Content-Type: application/json

Body:
{
  "click_id": "abc123",
  "event_name": "install",
  "payout": 2.50,
  "revenue": 5.00,
  "currency": "USD"
}

S2S token comes from your account manager — separate from your publisher API key.

Acceptable postback receiver endpoints (for advertiser-side firing)

EndpointConvention
GET /pb?clickid=...&event=...&payout=...Generic (always returns 200)
GET /postbacks?clickid=...Alias for /pb
GET /acquisition?clickid=...&security_token=...Trackier-compatible
GET /adjust/event?app_token=...&event_token=...&s2s=1Adjust-compatible
POST /skan/postbackApple SKAN raw
All postback receivers return HTTP 200

Per industry convention — networks don't retry 4xx responses. We always 200 and store the status (attributed / duplicate / rejected) on our side.

16. Testing your integration

Open your tracking link in a browser

Expect 302 redirect to advertiser's destination URL. If you see 404 "tracking link is no longer active" — the campaign token is wrong or campaign is paused.

Check the click was recorded

In your portal → Clicks, the click should appear within ~1 second. Verify publisher_click_id matches what you sent.

Fire a test conversion

If you have admin access, use the Campaign Link Tester at /test-link — supports geo bypass via security_token.

Otherwise: ask your account manager to fire a test postback. They can use:

https://track.apogeemobi.com/pb
  ?clickid=YOUR_TEST_CLICK_ID
  &event=install
  &payout=1

Verify postback received on your side

Your tracker should log the incoming request with clickid matching the original.

Check reports

Your portal → Reports → By Day should show the conversion within a few minutes.

17. Troubleshooting

SymptomLikely causeFix
Click returns HTTP 404 "tracking link is no longer active" Wrong CAMPAIGN_TOKEN or campaign paused/archived Re-copy URL from portal. Confirm campaign status with AM.
Click returns 302 but goes to a different URL than expected Your country/region/device is blocked by campaign targeting — redirected to geo_fallback_url Check campaign's allowed countries/regions/devices. Use a matching IP / UA to test.
Click returns 200 GEO_BLOCKED or 200 GEO_BLOCKED:DEVICE Geo/device targeting filter rejected the click. No fallback URL configured. Use traffic that matches campaign's allowed_countries / allowed_devices.
Mobile click returns 200 CLICK_BLOCKED "empty advertising_id" Mobile campaign requires advertising_id param Send advertising_id=<GAID or IDFA> on every mobile click.
Postback never arrives even though click was recorded (a) Conversion never fired on advertiser side, (b) lookback window expired, (c) clickid mismatch Confirm advertiser fired postback. Default click lookback is 7 days — extend in campaign settings if your traffic converts slowly.
Postback arrives but {click_id} is literal text, not substituted Your postback URL has typo (e.g. {clikid}) OR you're using a non-supported macro Check macro name against Section 10. Macros are case-sensitive.
Postback fires multiple times for the same conversion Advertiser fired multiple times — second one is status=duplicate Filter on {status}=attributed in your tracker, ignore duplicates.
Your sub1 / source value is empty in postback You didn't send it on the click — only echoed back if originally provided Add &sub1=source_name to every click URL.
