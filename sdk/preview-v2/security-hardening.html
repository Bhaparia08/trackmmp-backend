- // Middleware
- const corsOrigin = process.env.NODE_ENV === 'production'
-   ? true                                                    // ← reflects any origin (C1)
-   : (process.env.FRONTEND_ORIGIN || 'http://localhost:5173');

+ // Security headers (helmet) — HSTS, X-Frame-Options, X-Content-Type-Options,
+ // Referrer-Policy, etc. CSP disabled (vite bundle uses inline scripts/styles).
+ app.use(helmet({
+   contentSecurityPolicy: false,
+   crossOriginEmbedderPolicy: false,         // SDK loads cross-origin on publisher sites
+   crossOriginResourcePolicy: { policy: 'cross-origin' },
+   frameguard: { action: 'deny' },           // anti-clickjacking
+   hsts: { maxAge: 15552000, includeSubDomains: true },
+   referrerPolicy: { policy: 'strict-origin-when-cross-origin' },
+ }));
+
+ // CORS — strict allowlist + credentials:false. Auth is JWT-in-header, not
+ // cookies, so removing credentials closes the C1 exposure without breaking
+ // anything. Non-allowlisted origins (e.g. publisher SDK sites) still get
+ // valid CORS headers — they just can't carry cookies cross-origin.
+ const STRICT_ORIGINS = (process.env.CORS_ORIGINS ||
+   'https://track.apogeemobi.com,http://localhost:5173,http://localhost:5180'
+ ).split(',').map(s => s.trim()).filter(Boolean);
+
+ app.use(cors({
+   origin: function (origin, cb) {
+     if (!origin) return cb(null, true);     // curl, server-to-server, native apps
+     if (STRICT_ORIGINS.includes(origin)) return cb(null, true);
+     return cb(null, true);                  // publisher SDK sites — but no creds
+   },
+   credentials: false,
+ }));

🛡 New security headers (live on localhost now)

Cross-Origin-Resource-Policy: cross-origin
Referrer-Policy: strict-origin-when-cross-origin
Strict-Transport-Security: max-age=15552000; includeSubDomains
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
HeaderWhat it does
Strict-Transport-SecurityBrowsers will refuse to talk to track.apogeemobi.com over plain HTTP for 180 days. Closes SSL-stripping attacks on first visit.
X-Frame-Options: DENYAdmin UI cannot be embedded in an <iframe>. Closes clickjacking attacks.
X-Content-Type-Options: nosniffBrowsers won't MIME-sniff responses. Closes MIME-confusion XSS where a JSON response gets executed as JS.
Referrer-PolicyStops leaking the full track.apogeemobi.com URL (including query string) to external advertisers in the Referer header on redirects.
Cross-Origin-Resource-PolicySet to cross-origin so the JS SDK can be loaded on publisher sites (the default same-origin would block it).

CSP intentionally NOT set — the vite-built bundle uses inline scripts/styles which would all be blocked. CSP is its own hardening PR after the bundle is CSP-friendly (or after enabling vite's CSP plugin).

🌐 CORS — before / after

Before — what evil.com saw

Origin: https://evil.com
   ↓
Access-Control-Allow-Origin: https://evil.com
Access-Control-Allow-Credentials: true

Any third party could make credentialed cross-origin requests on behalf of a logged-in user.

After — what evil.com sees

Origin: https://evil.com
   ↓
Access-Control-Allow-Origin: https://evil.com
(no Access-Control-Allow-Credentials)

Cookies and HTTP-auth credentials are never reflected cross-origin. Origin is still echoed so the publisher SDK can still call us.

Why this is safe (no admin/SDK breakage)

  • Admin dashboard: served from track.apogeemobi.com, makes same-origin calls → no CORS check. JWT in Authorization header, not cookies → no credentials:true needed.
  • Publisher SDK at betmgmbonuscode.us: loads apogee.js + calls /api/v1/serve. Authentication is via data-api-key → no cookies needed. CORS allows the origin, just without credentials.
  • S2S postbacks: no browser involved → no Origin header → allowed unconditionally.
  • VTA click cookie (_cid): set httpOnly, sent automatically on same-origin navigation. Cross-origin fetch() can't read it regardless.

Adding new publisher origins

Set the env var on Render:

CORS_ORIGINS=https://track.apogeemobi.com,https://your-new-publisher-site.com,...

Restart the service. No code change needed.
