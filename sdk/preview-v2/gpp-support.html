Preview: GPP (Global Privacy Platform) support — Build #3

Built locally — awaiting deploy. Future-proofs the platform against the fragmenting US state privacy law landscape (CCPA, VCDPA, CPA, CTDPA, UCPA, etc.) by adopting the IAB Tech Lab's Global Privacy Platform — one wire format that carries all jurisdictional consent strings in one URL parameter.

📋 Why GPP?

In 2026, every US state is rolling its own privacy law. Publishers need ONE consent format that carries them all.

JurisdictionLawGPP Section ID
EUTCF v2.2Section 2
US NationalUSNAT (IAB Multi-State Privacy Agreement)Section 7
CaliforniaCCPA / CPRASection 8 (USCA)
VirginiaVCDPASection 9 (USVA)
ColoradoCPASection 10 (USCO)
UtahUCPASection 11 (USUT)
ConnecticutCTDPASection 12 (USCT)

A single GPP string like DBABMA~CPXxRf...~BAAAAAABA.QA can carry the user's consent for any combination of these. The publisher's CMP encodes; we just pass it through.

What we're not doing in this PR: parsing the GPP string. Parsing requires the full IAB GPP encoder/decoder library (variable-length integer encoding, etc.) — that lives publisher-side. Our job is to capture, store, and echo back. The result: zero risk of misinterpreting consent.

🔌 What it adds

1. New URL parameter on /track/click

https://track.apogeemobi.com/track/click/{token}
  ?pid=38
  &clickid=abc123
  &consent=accepted              (existing — TCF v2.2 single jurisdiction)
  &gpp=DBABMA~CPXxRf...~BAAAAA   (NEW — multi-jurisdiction GPP)

Either, both, or neither can be sent. Backward compatible — publishers that don't send GPP continue working exactly as before.

2. New DB column

ALTER TABLE clicks ADD COLUMN gpp_string TEXT;

Nullable, capped at 4096 chars on input. Typical real-world GPP strings are 200-1500 chars.

3. New postback macros (4 added)

MacroResolves to
{consent}TCF v2.2 consent string (alias)
{consent_state}Same — preferred name
{gpp}URL-encoded GPP string (alias)
{gpp_string}Same — preferred name

Works in both the click-redirect URL substitution (track.js inline map) AND the outbound postback URL substitution (utils/macroReplace.js).

4. OpenAPI updated

The spec at /api/v1/openapi.yaml now documents the consent and gpp query parameters under public-surfaces description.
