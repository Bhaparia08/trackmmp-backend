Comma-separated values. Allow = only this traffic passes (empty = all). Block = this traffic is rejected. Blocked clicks redirect to the Geo Fallback URL above (or return GEO_BLOCKED if none set). Unknown values (e.g. city when only country GeoIP is loaded) skip enforcement on that lever.

  4. NEW: allowed_cities / blocked_cities
  5. NEW: allowed_devices (existed but unenforced) / blocked_devices

Helper logic

function targetingPasses(actual, allowedCsv, blockedCsv) {
  const v = String(actual || '').trim().toLowerCase();
  if (!v) return true;                      // unknown value → skip enforcement
  const parse = (csv) => {
    const raw = String(csv || '').trim().toLowerCase();
    if (!raw || raw === 'all') return [];   // empty / 'all' → no filter
    return raw.split(',').map(s => s.trim()).filter(Boolean);
  };
  const allowed = parse(allowedCsv);
  const blocked = parse(blockedCsv);
  if (allowed.length > 0 && !allowed.includes(v)) return false;
  if (blocked.length > 0 &&  blocked.includes(v)) return false;
  return true;
}

Action on miss

  • If geo_fallback_url set → 302 redirect there (with publisher params forwarded — same as existing country block)
  • If no fallback → INSERT click with status='geo_blocked' + return GEO_BLOCKED:<LEVER> (e.g. GEO_BLOCKED:CITY)

Response body change for blocked clicks: GEO_BLOCKED → GEO_BLOCKED:<LEVER>. Country-block path still returns plain GEO_BLOCKED. If any downstream consumer parses this string for retries, the change is additive (still starts with GEO_BLOCKED).

Prerequisites for full feature to work on prod

  1. If you want city + region enforcement to actually fire on prod, the Render service needs GeoLite2-City.mmdb (free from MaxMind, ~70 MB). Without it, lookupGeo() returns empty region/city → those levers are silently skipped (correctly — same convention as country handling 'XX'). Country, device, OS, browser targeting will work fine without the City DB.
  2. Reports for region/city/browser are NOT in this PR — they're deferred to Phase A v2 so this stays a single reviewable change. Until then, you can query clicks directly for those slices.
  3. Platform-wide fraud rules (blocked_region, blocked_city, etc.) are also deferred. Per-campaign targeting works without them.

  3. cd backend && git add public/ for the new bundle
