Admitad payout currency always shows USD — even when the action is in EUR/GBP/INR

Found in routes/integrations.js fetchAdmitad. 1-file fix, ~5 lines. Backend only, no bundle rebuild needed.

1 · What's wrong today

Admitad's /advcampaigns response returns campaigns whose actions[] array can contain multiple actions, each with its own currency. Our fetcher correctly picks the highest-payout action (by numeric amount), but then throws away the action's currency and falls back to the campaign-level currency, which is often hard-coded to USD even when actions are denominated in other currencies.

Current code path in routes/integrations.js lines 591–647:

return allOffers.map(o => { let payout = 0; let payout_type = 'cpa'; const actions = Array.isArray(o.actions) ? o.actions : []; for (const act of actions) { const raw = String(act.payment_size || act.payout || '0'); const isPercent = raw.includes('%'); const parts = raw.replace('%', '').split('-'); const amount = parseFloat(parts[parts.length - 1] || parts[0] || '0') || 0; const type = isPercent ? 'revshare' : normPayoutType(act.type || 'cpa'); if (amount > payout) { payout = amount; payout_type = type; // 🐛 act.currency is NEVER captured here } } ... return { payout, payout_type, currency: o.currency || 'USD', // ← campaign-level — wrong when actions are in other currencies ... }; });

3 · The proposed fix (~5 lines)
return allOffers.map(o => { let payout = 0; let payout_type = 'cpa'; let payout_currency = null; // NEW — track winning action's currency const actions = Array.isArray(o.actions) ? o.actions : []; for (const act of actions) { const raw = String(act.payment_size || act.payout || '0'); const isPercent = raw.includes('%'); const parts = raw.replace('%', '').split('-'); const amount = parseFloat(parts[parts.length - 1] || parts[0] || '0') || 0; const type = isPercent ? 'revshare' : normPayoutType(act.type || 'cpa'); if (amount > payout) { payout = amount; payout_type = type; payout_currency = act.currency || act.payment_currency || null; } } ... return { payout, payout_type, currency: payout_currency || o.currency || 'USD', // action wins, falls back to campaign ... }; });
Why this is safe: when Admitad doesn't expose act.currency (older campaigns, simple flat offers), we fall back to o.currency which is today's behavior. So existing well-formed offers won't change. Only offers WITH explicit action-level currency get the corrected value.
